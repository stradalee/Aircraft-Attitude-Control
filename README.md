# 🛩️ Aircraft Attitude Control with LQR
 
행렬 기반 상태 공간 모델과 LQR(Linear Quadratic Regulator) 제어를 활용한
항공기 자세(Attitude) 제어 시뮬레이터입니다.
 
---

| 항목       | 내용                               |
| -------- | -------------------------------- |
| 목표       | 항공기의 피치·롤·요 축 자세를 최적 제어로 안정화     |
| 제어 기법    | LQR (Linear Quadratic Regulator) |
| 수치 적분    | 오일러 전진법 (Euler Forward Method)   |

## 🧮 제어 이론 개요
 
### 상태 공간 모델
 
시스템은 6차원 상태 벡터로 표현됩니다 (단위: rad, rad/s):
 
```
x = [pitch, pitch_rate, roll, roll_rate, yaw, yaw_rate]ᵀ
```
 
상태 방정식:
 
```
ẋ = Ax + Bu
```
 
- `A` : 6×6 시스템 행렬. 피치·롤·요 각 축을 2차 감쇠 진동계
  (`θ̈ = -wₙ²θ - 2ζwₙθ̇`)로 독립 모델링했습니다.
  - 피치: wₙ = 2.0 rad/s, ζ = 0.7
  - 롤  : wₙ = 1.5 rad/s, ζ = 0.7
  - 요  : wₙ = 1.0 rad/s, ζ = 0.5
- `B` : 6×3 입력 행렬. 엘리베이터/에일러론/러더가 각 축 각속도에 독립적으로 작용합니다.
- 출력 방정식(`y = Cx + Du`)은 전체 상태를 그대로 피드백에 사용하는
  완전 상태 피드백(full-state feedback) 구조라 `C`, `D`는 별도로 두지 않았습니다.
> 이전 버전은 롤 축 결합 항(`A[1,2]`, `A[3,2]`)에 물리적으로 근거 없는 값이 들어가
> 개루프 시스템에 양의 실수부 고유값(λ ≈ +0.316, 불안정 극)이 존재했습니다.
> 현재는 각 축을 물리적으로 의미 있는 2차 감쇠계로 교체해 개루프부터 안정적입니다.
 
### LQR 제어기 설계
 
연속시간 리카티 방정식(CARE)을 풀어 최적 제어 이득 `K`를 계산합니다:
 
```
AᵀP + PA - PBR⁻¹BᵀP + Q = 0
K = R⁻¹BᵀP
```
 
피드백 제어 법칙:
 
```
u = -Kx
```
 
가중 행렬:
- `Q = diag(10, 1, 10, 1, 10, 1)` — 각도 오차에 각속도보다 10배 높은 페널티
- `R = diag(1, 1, 1)` — 세 조종면 입력에 동일한 비중
LQR을 풀기 전, 시스템이 완전 제어가능(controllable)한지 제어가능성 행렬의
랭크로 검증하고, 계산된 `K`로 만든 폐루프 행렬 `A - BK`의 모든 고유값이
음의 실수부를 갖는지(즉 실제로 안정한지) 확인합니다. 둘 중 하나라도
실패하면 예외를 발생시킵니다.
 
> `np.matrix`는 NumPy 1.25부터 deprecated이며 `*` 연산자의 동작(행렬곱/원소곱)이
> 피연산자 타입에 따라 달라질 수 있어, `np.linalg.solve`와 `@` 연산자 기반으로
> 변경했습니다.

## 🚀 실행 방법
 
### 설치
 
```bash
pip install numpy scipy
```
 
### 실행
 
```bash
python Aircraft_Attitude_Control.py
```
 
### 출력 예시
 
```
==================================================
초기 상태 (도): [5. 0. 3. 0. 2. 0.]
최종 상태 (도, t=10.0s): [-0. -0.  0. -0.  0. -0.]
0.1° 이내 수렴 시간: 3.08s (308스텝)
==================================================
```
 
초기 자세 오차(피치 5°, 롤 3°, 요 2°)가 LQR 제어에 의해 약 3초 안에
0.1° 이내로 수렴합니다.
 
> 입력 단위는 항상 도(°)입니다. 시뮬레이션 내부에서 라디안으로 변환해
> 적분하고, 출력 시 다시 도 단위로 환산합니다 (이전 버전은 도 단위
> 입력값을 변환 없이 라디안 시스템에 그대로 대입하는 단위 불일치가
> 있었습니다).
